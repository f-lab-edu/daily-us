# 피드 성능 테스트 01

`celebrity의 게시글 작성`과 `다수 사용자의 피드 조회`를 테스트한다.

## 1. 데이터

[`docs/10_Resource/sample_data_seed_2.sql`](../../10_Resource/sample_data_seed_2.sql)

- 전체 사용자 수: `140,001`
- celebrity 계정: `userId = 1`
- celebrity 팔로워 수: `100,000`
  - 팔로워 구간: `userId 2..100001`
- 일반 조회 사용자 수: `40,000`
  - 일반 사용자 구간: `userId 100002..140001`
- celebrity 초기 게시글 수: `1,000`
- follower pool 게시글 수: `10,000명 x 5건`
- general pool 게시글 수: `20,000명 x 3건`

## 2. 시나리오

### 2.1 사용자 유형 비율

- follower reader: `70%`
- general reader: `30%`

### 2.2 요청 비율

- `GET /api/v1/posts?size=10`: `80%`
- `POST /api/v1/posts`: `20%`

### 2.3 쓰기
- 쓰기는 celebrity user만 수행한다.

### 2.3 조회

- follower reader는 로그인 후 자신의 피드를 조회한다.
- general reader는 일반 사용자로 로그인 후 자신의 피드를 조회한다.

## 4. nGrinder 스크립트

[`script/celebrity_feed_read_write_mix.groovy`](script/celebrity_feed_read_write_mix.groovy)

- reader는 `follower 70% / general 30%` 비율을 유지한다.
- 테스트는 읽기(8), 쓰기(2)로 분리한다.
