# 피드 성능 테스트 01 결과 분석

## 1. 개요

celebrity 게시글 작성과 다수 사용자의 피드 조회가 동시에 발생하는 상황

## 2. 테스트 대상

- 피드 조회 API: `GET /api/v1/posts`
- 게시글 작성 API: `POST /api/v1/posts`
- 부하 스크립트: `script/celebrity_feed_read_write_mix.groovy`

## 3. 테스트 환경

- 부하 도구: `nGrinder`
- 실행 시간: 약 `5분`
- 테스트 조건: `24 vUsers`, `50 vUsers`
- celebrity 계정: `userId = 1`
- celebrity 팔로워 수: `100,000`

### 요청 비율

- follower reader: `70%`
- general reader: `30%`
- feed read 요청: `80%`
- celebrity post write 요청: `20%`

## 6. TPS

- 테스트 중 TPS가 `0`에 가까운 수준까지 떨어지는 구간이 여러 번 확인

## 7. Celebrity Fan-out 부하

현재 피드 캐시 갱신 방식은 게시글 생성 이벤트를 기준으로 fan-out을 수행
팔로워 `100,000명`을 가진 celebrity 게시글로 인해 내부 fan-out 대상은 크게 증가
초당 `64~90건` 수준의 write 요청이지만, 처리 시 초당 약 `640만~900만`개의 피드 캐시 반영 대상으로 증폭
