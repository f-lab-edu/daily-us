## 4. 공통 정책

### 4.1 인증/인가
- 그룹 목록 조회와 그룹 상세 조회는 로그인 사용자 기준 기능으로 해석한다.
- 그룹 생성, 참여, 나가기, 해체는 로그인 사용자만 가능하다.
- 그룹 해체는 그룹장만 가능하다.
- 그룹 나가기는 일반 그룹원만 가능하다.

### 4.2 삭제 정책
- 삭제는 소프트 삭제를 기본 전제로 본다.
- 해체된 그룹은 목록 조회와 상세 조회에서 제외한다.
- 그룹 해체 시 그룹원 데이터와 그룹 관련 조회 데이터도 함께 정리되어야 한다.

### 4.3 정렬/페이징 정책
- 그룹 목록은 순서 상관 없이 10개씩 페이징 처리한다.
- 본인 그룹 조회는 별도 정렬 기준이 없다.
- 그룹 랭킹은 월 단위로 산정한다.

### 4.4 데이터 제약 조건
- 그룹 참여는 동일 사용자 기준 같은 그룹에 중복 가입할 수 없다.
- 그룹 나가기는 그룹에 가입한 사용자만 가능하다.
- 그룹 해체는 그룹장만 가능하다.
- 그룹원 수는 0 미만이 될 수 없다.
- 그룹 최대 인원은 100명으로 제한한다.
- 사용자당 가입 가능 그룹 수는 최대 200개로 제한한다.

## 5. 기능 상세 정의

### 5.1 그룹 목록 조회

#### 목적
- 사용자가 개설되어 있는 그룹 목록을 조회한다.

#### 노출 대상
- 삭제되지 않은 그룹이 대상이다.

#### 입력 조건
- 페이지네이션 정보

#### 처리 규칙
- 그룹 목록은 순서 상관 없이 제공한다.
- 한 번에 10건씩 페이징 처리한다.
- 그룹 목록에는 그룹명, 그룹 대표 이미지가 포함된다.

#### 예외
- 조회 가능한 그룹이 없으면 빈 목록을 반환한다.

### 5.2 그룹 상세 조회

#### 목적
- 사용자가 특정 그룹의 상세 정보를 조회한다.

#### 입력 항목
- `groupId`

#### 처리 규칙
- 그룹명, 인원 수, 그룹장, 그룹 설명 또는 목적을 조회한다.
- 삭제된 그룹은 조회 대상에서 제외한다.

#### 예외
- 존재하지 않거나 삭제된 그룹이면 실패 응답을 반환한다.

### 5.3 그룹 생성

#### 목적
- 사용자가 새 그룹을 생성한다.

#### 입력 항목
- 그룹명
- 그룹 소개
- 그룹 대표 이미지

#### 처리 규칙
- 로그인 사용자만 그룹을 생성할 수 있다.
- 생성자는 그룹장(owner)이 된다.
- 그룹 생성 후 생성자는 해당 그룹의 구성원으로 포함되어야 한다.
- 생성된 그룹은 그룹 목록에 반영된다.

#### 데이터 반영
- 그룹 마스터 데이터를 생성한다.
- 생성자를 그룹 멤버로 등록한다.

#### 예외
- 입력 항목 미입력 시 생성할 수 없다.

### 5.4 그룹 참여

#### 목적
- 사용자가 기존 그룹에 참여한다.

#### 입력 항목
- `groupId`

#### 처리 규칙
- 로그인 사용자만 참여할 수 있다.
- 이미 가입한 그룹에는 중복 참여할 수 없다.
- 참여 후에는 그룹 랭킹과 구성원 정보를 조회할 수 있다.
- 참여 성공 시 본인 그룹 목록과 그룹 인원 수에 반영한다.

#### 데이터 반영
- `group_members`에 그룹 참여 레코드를 생성한다.

#### 예외
- 존재하지 않거나 삭제된 그룹에는 참여할 수 없다.
- 이미 가입한 그룹이면 중복 참여를 허용하지 않는다.

### 5.5 그룹 나가기

#### 목적
- 일반 그룹원이 자신이 속한 그룹에서 탈퇴한다.

#### 입력 항목
- `groupId`

#### 처리 규칙
- 로그인 사용자만 가능하다.
- 그룹에 가입한 사용자만 나갈 수 있다.
- 탈퇴 시 그룹원 수를 감소시키고, 탈퇴한 회원은 그룹원 목록에서 제외한다.
- 탈퇴 후 본인 그룹 목록에서도 제외한다.

#### 예외
- 가입하지 않은 그룹은 나갈 수 없다.
- 그룹장은 일반 탈퇴 대상이 아니며, 해체 기능을 사용해야 한다고 해석한다.

### 5.6 그룹 해체

#### 목적
- 그룹장이 자신이 만든 그룹을 해체한다.

#### 입력 항목
- `groupId`

#### 처리 규칙
- 그룹장만 해체할 수 있다.
- 그룹이 해체되면 그룹원, 랭킹, 그룹 데이터를 함께 삭제 처리한다.
- 해체된 그룹은 그룹 목록, 그룹 상세, 본인 그룹 목록에서 제외한다.

#### 예외
- 그룹장이 아니면 해체할 수 없다.
- 존재하지 않거나 이미 해체된 그룹은 해체할 수 없다.

### 5.7 그룹원 조회

#### 목적
- 사용자가 특정 그룹의 구성원 목록을 조회한다.

#### 입력 항목
- `groupId`

#### 처리 규칙
- 그룹에 속한 사용자 목록을 조회한다.
- 각 구성원에는 사용자 식별자, 닉네임, 프로필 이미지가 포함된다.
- 탈퇴한 사용자는 목록에서 제외한다.

#### 예외
- 존재하지 않거나 삭제된 그룹이면 조회할 수 없다.

### 5.8 그룹 내 랭킹 조회

#### 목적
- 사용자가 그룹 화면에서 그룹원 랭킹을 조회한다.

#### 입력 항목
- `groupId`
- 조회 기준 월

#### 처리 규칙
- 랭킹은 월 단위로 산정한다.
- 순위, 사용자명, 활동 횟수를 제공한다.
- 활동 횟수는 해당 월의 게시글 업로드 수를 기준으로 계산한다.
- 그룹원만 랭킹 집계 대상이 된다.

#### 예외
- 존재하지 않거나 삭제된 그룹이면 조회할 수 없다.
- 기준 월이 유효하지 않으면 실패 처리한다.
