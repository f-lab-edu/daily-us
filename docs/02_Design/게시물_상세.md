## 4. 공통 정책

### 4.1 인증/인가
- 게시글 조회, 작성, 수정, 삭제, 좋아요는 로그인 사용자만 가능하다.
- 게시글 수정/삭제는 게시글 작성자 본인만 가능하다.

### 4.2 삭제 정책
- 삭제는 소프트 삭제를 기본 전제로 본다.
- 게시글 삭제 시 연관 데이터도 함께 삭제 처리한다.
- 삭제된 게시글은 조회에서 제외한다.

### 4.3 정렬/페이징 정책
- 피드 게시글 목록은 작성 시간 기준 최신순 정렬한다.
- 피드 게시글 목록은 10개씩 페이징한다.
- 프로필 게시글 목록은 작성일 기준 최신순 정렬한다.
- 프로필 게시글 목록은 9개씩 페이징한다.

### 4.4 데이터 제약 조건
- 게시글은 이미지를 최소 1장 이상 첨부해야 한다.
- 게시글 첨부 이미지는 최대 10장까지 허용한다.
- 게시글 수정 후에도 이미지는 최소 1장 이상 유지해야 한다.
- 게시글 본문 최대 길이는 2,200자로 제한한다.
- 해시태그는 게시글당 최대 10개까지 허용한다.
- 해시태그 최대 길이는 30자로 제한한다.
- 동일 해시태그의 중복 입력은 허용하지 않는다.
- 게시글 좋아요는 사용자당 게시글별 1회만 가능하다.
- 게시글 좋아요 수는 0 미만이 될 수 없다.

## 5. 기능 상세 정의

### 5.1 게시글 목록 조회(피드)

#### 목적
- 사용자가 메인 피드에서 게시글 목록을 조회한다.

#### 노출 대상
- 로그인한 사용자만 피드 목록을 조회할 수 있다.
- 기본 노출 대상은 본인이 팔로우한 사용자 또는 본인이 속한 그룹 멤버의 게시글이다.
- 팔로우한 사용자와 소속 그룹이 모두 없으면 게시글을 랜덤 노출한다.

#### 입력 조건
- 페이지네이션 커서 또는 마지막 조회 시점 정보가 필요하다.

#### 처리 규칙
- 삭제되지 않은 게시글만 조회한다.
- 최신 작성순으로 정렬한다.
- 한 번에 10건 조회한다.
- 각 게시글에는 첨부 이미지, 작성자 정보, 본문, 좋아요 정보를 포함한다.

#### 예외
- 조회 대상 게시글이 없으면 빈 목록을 반환한다.

### 5.2 게시글 상세 조회

#### 목적
- 사용자가 선택한 게시글의 전체 내용을 조회한다.

#### 입력 조건
- `postId`

#### 처리 규칙
- 게시글 본문, 첨부 이미지, 작성자 정보, 좋아요 정보를 조회한다.
- 삭제된 게시글은 조회 대상에서 제외한다.

#### 예외
- 존재하지 않거나 삭제된 게시글이면 실패 응답을 반환한다.

### 5.3 게시글 작성

#### 목적
- 사용자가 이미지와 본문을 포함한 게시글을 생성한다.

#### 입력 항목
- 첨부 이미지 목록
- 본문

#### 처리 규칙
- 이미지를 첨부하지 않은 게시글은 작성할 수 없다.
- 여러 장의 이미지를 첨부할 수 있다.
- 본문은 비어 있을 수 있다.
- 게시글 생성 시 피드와 내 프로필 게시글 목록에 반영한다.
- 본문 내 해시태그가 존재하면 해시태그 저장 및 게시글 매핑을 생성한다.

#### 데이터 반영
- `posts` 테이블에 게시글을 생성한다.
- `post_images` 테이블에 첨부 이미지 레코드를 생성한다.
- 해시태그가 있으면 `hashtag`, `hashtag_posts`를 갱신한다.

#### 예외
- 이미지가 하나도 없으면 작성 실패 처리한다.
- 인증되지 않은 사용자는 작성할 수 없다.

### 5.4 게시글 수정

#### 목적
- 작성자가 기존 게시글의 이미지 또는 본문을 변경한다.

#### 입력 항목
- `postId`
- 수정할 이미지 목록
- 수정할 본문

#### 처리 규칙
- 작성자 본인만 수정할 수 있다.
- 이미지와 본문을 변경할 수 있다.
- 수정 후 게시글 상세와 피드에 반영한다.
- 해시태그가 변경되면 기존 매핑을 정리하고 신규 매핑으로 재구성한다.

#### 예외
- 본인 게시글이 아니면 수정할 수 없다.
- 존재하지 않거나 삭제된 게시글은 수정할 수 없다.
- 이미지 필수 정책을 유지하므로 수정 후에도 이미지가 최소 1장 이상이어야 한다.

### 5.5 게시글 삭제

#### 목적
- 작성자가 게시글을 삭제한다.

#### 입력 항목
- `postId`

#### 처리 규칙
- 작성자 본인만 삭제할 수 있다.
- 게시글 삭제 시 연관 데이터도 함께 삭제 처리한다.
- 삭제된 게시글은 피드, 프로필 게시글 목록, 상세 조회에서 제외한다.
- 활동 집계 데이터는 삭제 반영 기준을 동일하게 적용해야 한다.

#### 예외
- 본인 게시글이 아니면 삭제할 수 없다.
- 존재하지 않거나 이미 삭제된 게시글은 삭제할 수 없다.

### 5.6 게시글 좋아요

#### 목적
- 사용자가 게시글에 좋아요를 추가한다.

#### 입력 항목
- `postId`

#### 처리 규칙
- 로그인 사용자만 좋아요할 수 있다.
- 사용자당 하나의 게시글에 한 번만 좋아요할 수 있다.
- 좋아요 성공 시 `post_likes` 생성 및 `posts.like_count`를 반영한다.

#### 예외
- 이미 좋아요한 게시글이면 중복 좋아요를 허용하지 않는다.
- 존재하지 않거나 삭제된 게시글에는 좋아요할 수 없다.

### 5.7 게시글 좋아요 취소

#### 목적
- 사용자가 기존 게시글 좋아요를 취소한다.

#### 입력 항목
- `postId`

#### 처리 규칙
- 기존 좋아요 이력이 있을 때만 취소할 수 있다.
- 취소 성공 시 `post_likes` 삭제 및 `posts.like_count`를 반영한다.

#### 예외
- 좋아요 이력이 없으면 취소할 수 없다.
