## 4. 공통 정책

### 4.1 인증/인가
- 댓글 조회, 작성, 수정, 삭제, 좋아요는 로그인 사용자만 가능하다.
- 댓글 수정/삭제는 댓글 작성자 본인만 가능하다.

### 4.2 삭제 정책
- 삭제는 소프트 삭제를 기본 전제로 본다.
- 삭제된 댓글은 목록 조회에서 제외한다.
- 상위 댓글 삭제 시 답글도 함께 삭제하는 것으로 현재 문서에 정리한다.

### 4.3 정렬/페이징 정책
- 댓글 목록은 생성 시각 기준 정렬을 기본으로 한다.
- 답글 목록도 동일하게 생성 시각 기준 정렬을 적용한다.

### 4.4 데이터 제약 조건
- 댓글 내용은 비어 있을 수 없다.
- 댓글 최대 길이는 500자로 제한한다.
- 답글은 부모 댓글이 존재할 때만 작성할 수 있다.
- 답글은 동일한 게시글의 댓글에만 작성할 수 있다.
- 답글은 1단계까지만 허용한다.
- 댓글 좋아요는 사용자당 댓글별 1회만 가능하다.
- 댓글 좋아요 수는 0 미만이 될 수 없다.

## 5. 기능 상세 정의

### 5.1 댓글 목록 조회

#### 목적
- 사용자가 게시글 상세 화면에서 댓글 목록을 조회한다.

#### 노출 대상
- 선택한 게시글에 연결된 삭제되지 않은 댓글과 답글이 대상이다.

#### 입력 조건
- `postId`
- 페이지네이션 커서 또는 마지막 조회 시점 정보

#### 처리 규칙
- 게시글 기준 댓글 목록을 조회한다.
- 각 댓글에는 작성자 정보, 내용, 작성 시각, 좋아요 수, 로그인 사용자 좋아요 여부를 포함한다.
- `parent_id`를 기준으로 일반 댓글과 답글을 구분한다.
- 삭제된 댓글은 목록에서 제외한다.

#### 예외
- 존재하지 않거나 삭제된 게시글이면 댓글 조회를 실패 처리한다.

### 5.2 댓글 작성

#### 목적
- 사용자가 게시글에 일반 댓글을 작성한다.

#### 입력 항목
- `postId`
- 댓글 내용

#### 처리 규칙
- 로그인 사용자만 작성할 수 있다.
- 내용이 없는 댓글은 등록할 수 없다.
- 등록 후 댓글 목록에 즉시 반영한다.
- 일반 댓글은 `parent_id`를 `NULL`로 저장한다.

#### 데이터 반영
- `comments` 테이블에 댓글 레코드를 생성한다.

#### 예외
- 빈 내용은 허용하지 않는다.
- 존재하지 않거나 삭제된 게시글에는 댓글을 작성할 수 없다.

### 5.3 답글 작성

#### 목적
- 사용자가 기존 댓글에 대한 답글을 작성한다.

#### 입력 항목
- `postId`
- 댓글 내용
- `parentId`

#### 처리 규칙
- 로그인 사용자만 작성할 수 있다.
- 내용이 없는 답글은 등록할 수 없다.
- `parentId`는 존재하는 댓글이어야 한다.
- 부모 댓글이 속한 게시글과 `postId`가 동일해야 한다.
- 등록 후 댓글 목록에 즉시 반영한다.

#### 데이터 반영
- `comments` 테이블에 `parent_id`를 포함한 답글 레코드를 생성한다.

#### 예외
- 빈 내용은 허용하지 않는다.
- 존재하지 않거나 삭제된 부모 댓글에는 답글을 작성할 수 없다.
- 다른 게시글의 댓글을 부모로 지정할 수 없다.

### 5.4 댓글 수정

#### 목적
- 댓글 작성자가 본인 댓글을 수정한다.

#### 입력 항목
- `commentId`
- 수정 내용

#### 처리 규칙
- 작성자 본인만 수정할 수 있다.
- 수정 내용은 비어 있을 수 없다.
- 수정 후 목록에 반영하고 수정 여부를 표시한다.

#### 예외
- 본인 댓글이 아니면 수정할 수 없다.
- 존재하지 않거나 삭제된 댓글은 수정할 수 없다.
- 빈 내용으로 수정할 수 없다.

### 5.5 댓글 삭제

#### 목적
- 댓글 작성자가 본인 댓글을 삭제한다.

#### 입력 항목
- `commentId`

#### 처리 규칙
- 작성자 본인만 삭제할 수 있다.
- 삭제된 댓글은 목록에서 제거한다.
- 댓글 좋아요 데이터도 함께 정리한다.
- 답글이 있는 댓글 삭제 시 하위 댓글도 함께 삭제한다.

#### 예외
- 본인 댓글이 아니면 삭제할 수 없다.
- 존재하지 않거나 이미 삭제된 댓글은 삭제할 수 없다.

### 5.6 댓글 좋아요

#### 목적
- 사용자가 댓글에 좋아요를 추가한다.

#### 입력 항목
- `commentId`

#### 처리 규칙
- 로그인 사용자만 가능하다.
- 사용자당 하나의 댓글에 한 번만 좋아요할 수 있다.
- 좋아요 성공 시 `comment_likes` 생성 및 `comments.like_count`를 반영한다.

#### 예외
- 이미 좋아요한 댓글이면 중복 좋아요를 허용하지 않는다.
- 존재하지 않거나 삭제된 댓글에는 좋아요할 수 없다.

### 5.7 댓글 좋아요 취소

#### 목적
- 사용자가 기존 댓글 좋아요를 취소한다.

#### 입력 항목
- `commentId`

#### 처리 규칙
- 기존 좋아요 이력이 있을 때만 취소할 수 있다.
- 취소 성공 시 `comment_likes` 삭제 및 `comments.like_count`를 반영한다.

#### 예외
- 좋아요 이력이 없으면 취소할 수 없다.
